Loom · Reference

Build visual stories,
frame by frame.

Loom is a browser-based storyboard canvas. Arrange scene frames freely in 2D space, link them with strings to show narrative flow, add text, images, and links to each scene, then export your board as a shareable image or editable project file.

Getting Started

Open index.html in any modern browser and you're ready. Nothing installs, nothing syncs — your board lives entirely in your browser's local storage.

Your first storyboard

1
Create a frame
Press N or click New Frame in the top bar. A frame appears in the center of the canvas — it's your first scene card.
2
Name it
Click the frame's title field and type the scene name. The same field appears in the Inspector panel on the right.
3
Add content
Use the toolbar at the bottom of each frame to add a Text block, Image (from your device), or a Link.
4
Arrange and connect
Drag frames into position. Drag the pin icon ◎ from one frame header to another to draw a narrative connection.
5
Export
Click Export → Project to save a .json you can reopen later, or Export → Storyboard for a flattened .png snapshot.
💾

Loom auto-saves to localStorage after every action, so closing and reopening the tab will restore your work — as long as you're using the same browser.

The Canvas

The canvas is an infinite 2D world. Your frames float in it at any position. Navigate with pan and zoom; the grid and world origin orb help you stay oriented.

Panning (moving around)

Two gestures move the camera without interacting with frames:

  • Ctrl + Drag on any empty area of the canvas.
  • Middle-click drag anywhere on the canvas (including on frames).
💡

Ctrl + Drag on a frame moves the frame itself with snap-alignment, not the camera. Hold Ctrl while dragging a frame to engage snapping mode.

Zooming

Scroll the mouse wheel (or trackpad pinch) anywhere on the canvas to zoom in or out. The zoom level is shown as a percentage in the top-right corner of the top bar.

  • F — Zoom to fit all visible frames into view. Frames belonging to hidden groups are excluded.
  • 0 — Reset to 100 % zoom centered on the world origin.
  • Zoom range: 15 % – 250 %.

World origin

The point (0, 0) in world space is marked by a subtle glowing orb. Ctrl + Drag snap alignment and X/Y inspector values are all measured relative to this origin. Pressing 0 centers the camera on it.

Grid

The background shows a fine dot grid (32 px) with larger major marks (160 px). The grid moves with the camera to maintain visual orientation as you pan and zoom.

Box selection (marquee)

Click and drag on any empty area of the canvas (without holding Ctrl) to draw a selection rectangle. Any frame whose bounding box overlaps the rectangle will be selected when you release. Hold Shift while drawing to add to an existing selection.

Select all

Press Ctrl + A to instantly select all visible frames. Frames that belong to hidden groups are excluded from this selection.

Frames

A frame is a scene card — the fundamental unit of a Loom storyboard. Each frame has a title, a color tag, a free-form body for content blocks, and optional notes in the Inspector.

Adding frames

Press N or click New Frame in the top bar. New frames appear near the center of the current view, cascaded slightly from the last one so they don't perfectly overlap. The title field is focused automatically so you can name it immediately.

Moving frames

Drag the frame header (the bar containing the title and number) or any empty area of the frame body to reposition it freely.

Snap-aligned dragging (Ctrl + Drag)

Hold Ctrl while dragging a frame to engage snap mode. The frame's left edge, right edge, horizontal center, top edge, bottom edge, and vertical center will all snap to matching edges and centers of other frames — and to the world origin axes — within an 8-pixel threshold. Accent-colored guide lines appear when a snap is active.

  • Snap works for both single-frame drag and multi-frame group drag.
  • For group drags, the anchor frame (the one you grabbed) is used for snap detection; all other selected frames move by the same offset.

Resizing frames

Drag the small resize handle in the bottom-right corner of any frame. Minimum size is 180 × 130 px. Exact dimensions are shown and editable in the Inspector while resizing.

Selecting frames

Click frame
Select a single frame; opens it in the Inspector.
Click frame row
Click a row in the Frame List (left panel) to select and pan to that frame.
Marquee drag
Drag on empty canvas to box-select multiple frames.
Shift + marquee
Add frames to the current selection without clearing it.
Ctrl + A
Select all visible frames (hidden group frames are excluded).
Escape
Clear the selection.

Moving multiple frames

Once two or more frames are selected, drag any one of them to move the entire group together. The Inspector shows a lock notice while a multi-selection is active.

💡

Ctrl + Drag on a group uses the dragged frame's edges for snap detection, then offsets all other selected frames by the same snapped delta.

Deleting frames

  • Select a frame and press Delete or Backspace.
  • Click the × next to a frame in the Frame List.
  • Click Delete Frame in the Inspector.
  • With multiple frames selected, Delete removes all of them at once.

Deleting a frame also removes any connections it participates in. This action is undoable.

Frame line

Each frame has a colored accent stripe — its frame line. Use the Inspector's Frame line selector to position it on the Left, Right, Top, Bottom, or hide it entirely (None). The stripe color always matches the frame's tag color.

Frame Content

Each frame can hold any number of content blocks — text, images, and links — in any order. Use the toolbar at the bottom of the frame to add blocks.

Text blocks

Click Aa · Text in the frame toolbar to add a text block. The block is a plain contenteditable area — type freely. In View Mode (or when a frame's preview is toggled on), text blocks render as Markdown: use **bold**, *italic*, headings, lists, and so on.

✍️

Write your script or shot notes in Markdown inside text blocks, then switch to View Mode (or use the per-frame preview toggle) to see them rendered cleanly for presentation.

Image blocks

Click ▢ · Image to open your device's file picker. Select any image file — it's embedded into the storyboard as a Base64 data URL, so the image travels with your project file without needing separate assets.

⚠️

Large images significantly increase the size of your .json project file and may slow down export. Consider resizing images before importing.

Link blocks

Click 🔗 · Link, paste or type a URL, and press Enter or click Add. The block displays the site's hostname and full URL. If you paste a URL without a protocol, Loom prepends https:// automatically. Press Escape to cancel without adding a link.

Reordering blocks

Hover over any block to reveal controls on its right edge. Drag the grip icon (☰) to reorder the block within its frame by dragging it above or below other blocks.

Deleting blocks

Hover over any block and click the × button that appears. The block is removed immediately (undoable).

Connections

Connections are the threads that link frames in a parent → child relationship, rendered as golden bezier strings with brass tack endpoints. They form a tree hierarchy visible in the Frame List.

Drawing a connection

Hover over or select any frame. A pin icon (◎) appears in the top-right of the header. Drag from that pin to another frame — release over the target to create the connection.

  • The string preview is amber while dragging freely.
  • It turns teal when hovering over a valid target.
  • It turns red when hovering over an invalid target.

Connection constraints

The connection model enforces a strict tree structure:

  • Each frame can have at most one parent. Connecting a frame that already has a parent is not allowed.
  • Cycles are prevented — you cannot connect a frame to one of its own ancestors.
  • You cannot create a duplicate connection between the same two frames.

When a drag lands on an invalid target, Loom shows a toast message explaining why the connection was rejected.

Group inheritance

When you draw a connection from a parent frame that belongs to a group, the child frame (and all of its descendants) automatically inherit that group membership. This keeps logical subtrees together without manual reassignment.

Cutting a connection

Click anywhere along the string thread to remove it. The thread highlights red when hovered to signal that clicking will cut it.

String layer (front / behind frames)

Right-click any string thread to open the context menu:

  • Send behind frames — the string renders underneath all cards.
  • Bring in front of frames — the string renders above all cards (default).
  • Apply to all strings — applies the same layer choice to every visible connection on the board at once (hidden-group connections are skipped).

Connections and hidden groups

Connections involving frames in a hidden group are not drawn on the canvas. They are preserved in the project data and reappear when the group is made visible again.

Frame List

The left panel's Frames section lists every frame in hierarchy order, organized into groups and root-level frames. A + button in the section header creates new groups.

Navigation

Click any row to select the frame and pan the canvas to center it in view. This is useful for jumping to a specific scene in a large board without scrolling manually.

Hierarchy display

Child frames are indented beneath their parents with an arrow connector symbol (↳). The indentation is dynamic — each level adds 12 px of left padding — and reflects the parent→child relationships drawn on the canvas. Frames without any connections appear as siblings at the root level.

Color dot and number

Each row shows a colored dot matching the frame's tag color and a zero-padded frame number reflecting the order frames were created (not their spatial position).

Dragging frames between groups

Drag any frame row and drop it onto a group header row to assign the frame (and its entire descendant subtree) to that group. Drag a frame row to the Frames section header to remove it from any group.

Deleting from the list

Hover a frame row to reveal a × delete button on the right. Clicking it removes the frame and all its connections — identical to pressing Delete with the frame selected on the canvas.

Groups

Groups let you cluster related frames together in the Frame List. They support collapsing, hiding, optional visual border highlights, drag-to-reorder, and a right-click context menu for quick actions.

Creating a group

Click the + button in the top-right corner of the Frames section header. A new group row appears named "New Group" with zero frames. Click the name field to rename it.

Adding frames to a group

Drag any frame row from the Frame List and drop it onto the target group row. The frame and all of its connected descendants move into the group together. A toast confirms how many frames were moved.

💡

You cannot drag a frame into a group it already belongs to — Loom will notify you with a toast instead.

Removing frames from a group

Drag any frame row to the Frames section header (the area above all groups). The frame and its descendants are unassigned from the group and returned to the ungrouped root level.

Collapsing and expanding

Click the chevron (▾) on the left of a group row to collapse or expand the group's frame rows in the Frame List. When collapsed, the frames remain fully visible on the canvas — collapsing is a list-view-only convenience, not a hide operation.

Hiding and showing group frames

Click the eye icon button on the group row to hide all frames in that group. Hidden frames:

  • Are removed from the canvas entirely (not just invisible — they don't receive clicks or participate in box-selection).
  • Have their connection strings hidden as well.
  • Are excluded from Zoom to Fit and Ctrl + A select-all.
  • Are excluded from the storyboard PNG export.
  • Remain collapsed in the Frame List with all interactive controls dimmed.

Click the eye icon again (it shows a slashed eye while hidden) to restore all frames in that group.

⚠️

Hiding a group also disables its border highlight and collapses it. Showing the group again re-expands it, but does not automatically re-enable the border.

Border highlight

Click the dashed-rectangle icon button on a group row to toggle a visual border around all frames in the group on the canvas. The border is a dashed SVG rect drawn behind the frames, sized to tightly wrap the group's bounding box with a small padding. It uses the group's accent color (or the default teal accent if none is set).

The border updates live as frames within the group are moved or resized. It is hidden automatically when the group is hidden.

Right-click context menu

Right-click any group row (while in Edit Mode and while the group is not hidden) to open a context menu with:

  • Hide Frames / Show Frames — same as the eye button.
  • Show Borders / Hide Borders — same as the border button.
  • Delete Group — removes the group and unassigns all its frames, without deleting the frames themselves.

Reordering groups

Hover a group row to reveal a grip handle (☰) on the left. Drag from the grip to reorder groups within the Frame List. Only groups can be reordered this way — frame rows within a group follow the connection hierarchy.

Deleting a group

Use the right-click context menu and choose Delete Group. The group is removed and all its frames are unassigned (returned to the ungrouped pool) but not deleted from the canvas. This is undoable.

Inspector

The Inspector panel on the right shows editable metadata for the currently selected frame. Selecting multiple frames locks the Inspector and shows a count badge instead.

Title

The frame's display name. Editable here or directly in the frame header on the canvas — the two stay in sync. Maximum 60 characters. In View Mode (or with per-frame preview enabled), the title renders as Markdown.

Tag color

Six preset swatches set the frame's accent color, which tints the border, glow, frame-line stripe, and pin button.

Position (X, Y) and Size (W, H)

The exact world-space coordinates of the frame's top-left corner, and its width and height in world pixels. All four fields are editable — type a value and press Enter or click away to apply. Minimum size is W: 180, H: 130.

These values update live while you drag or resize a frame.

Notes

A multi-line text area for director's notes, camera descriptions, dialogue, or any free-form annotation. Notes are Inspector-only metadata — they are not displayed inside the frame card on the canvas. In View Mode, notes render as Markdown.

Frame line

A segmented button row with five options: Left, Right, Up, Down, None. Controls which edge of the frame the colored accent stripe sits on. Defaults to Left for all new frames.

Delete Frame

The red Delete Frame button at the bottom of the Inspector removes the selected frame and all its connections. If multiple frames are selected, it deletes all of them.

Edit & View Modes

Loom has two distinct modes accessed from the mode dropdown in the top bar. Your last-used mode is remembered across sessions.

Edit Mode

Full editing is available: add, move, resize, and delete frames; manage blocks; draw and cut connections; manage groups; adjust the Inspector; undo and redo. This is the default mode when you first open Loom.

View Mode

Read-only presentation mode. Editing controls are hidden from the UI:

  • New Frame button, Clear button, and frame toolbars are hidden.
  • Block controls, resize handles, delete buttons, and pin icons are hidden.
  • The Inspector fields become read-only.
  • Undo and Redo are disabled.
  • Group action buttons are disabled.
  • Text blocks, titles, and notes render as Markdown — use this to show polished formatted content to collaborators.

Pan, zoom, frame selection, and canvas navigation all remain fully functional in View Mode.

📋

Use Import to load a .json project — Loom automatically opens imported files in View Mode, making it easy to share a board for review without exposing editing tools.

Switching modes

Open the mode dropdown (top-left of the top bar) and choose Edit Mode or View Mode. A dot indicator in the button turns teal in Edit Mode and amber in View Mode. A toast confirms the switch.

Frame Preview

In Edit Mode, each frame can independently switch to a Markdown preview — letting you check rendered content without switching the entire board to View Mode.

Per-frame preview toggle

Hover over or select any frame. An eye icon appears in the top-right corner of the frame header, next to the pin button. Click it to toggle that frame between edit view (raw text) and preview (rendered Markdown). The toggle state persists across re-renders while you're in the same session.

When a frame is in preview mode:

  • The frame toolbar (Text / Image / Link) is hidden.
  • The resize handle is hidden.
  • Block drag and delete controls are hidden.
  • Title, text blocks, and notes all render as Markdown.

Clicking the eye icon again restores the frame to its normal editable state.

Toggle all previews

Press Ctrl + Shift + P to toggle preview mode on or off for all visible frames at once. If any frames currently have preview active, the shortcut turns preview off for all of them. If none do, it turns preview on for all. A toast confirms the action.

💡

Per-frame previews are a session-only convenience — they are not saved to the project file and reset when you reload the page. To make Markdown rendering permanent and exportable, use View Mode.

Undo & Redo

Loom tracks your full editing history in memory. Undo and Redo buttons sit in the top bar and are available by keyboard shortcut in Edit Mode.

Shortcuts

KeysAction
Ctrl + ZUndo
Ctrl + YRedo
Ctrl + Shift + ZRedo (alternate)

What is and isn't tracked

The history system takes a snapshot of the entire board state after each meaningful action:

  • Tracked: adding/deleting frames, moving frames, resizing, changing colors, adding/removing blocks, drawing/cutting connections, frame line changes, clearing the board, reordering blocks, creating/deleting/renaming/reordering groups, toggling group borders, hiding/showing groups, assigning frames to groups.
  • Not tracked: camera position (pan/zoom), panel widths, group collapse state, per-frame preview state — these are intentionally preserved across undo/redo so the viewport and UI don't jump.

Debounced text edits

Typing in title, notes, group name, or text block fields doesn't create a new history entry on every keystroke. Changes are batched and committed after a 700 ms pause in typing. If you delete a frame immediately after typing, Loom flushes the pending text snapshot first so the type and delete are two separate undo steps.

History depth

Up to 60 history states are kept. Older states are discarded when the limit is reached.

⚠️

History lives in memory only and is lost when you close or refresh the tab. For long sessions, export a .json project as a manual checkpoint.

Import & Export

Loom can save your work as a portable JSON project file or render a flat PNG image of the complete board.

Export → Project (.json)

Downloads the complete board state as a .json file. The filename is generated from the project name with special characters stripped. This file contains:

  • All frames — positions, sizes, titles, colors, frame line settings, block content, and group assignments.
  • All connections between frames and their layer settings.
  • All groups — names, collapsed state, hidden state, border state, and sort order.
  • The current camera position and zoom level.
  • Panel layout widths and the project name.
💡

The project JSON is plain text and human-readable. You can inspect or batch-edit it with any text editor.

Export → Storyboard (.png)

Renders a flat PNG image of the board. The export only includes visible frames — frames belonging to hidden groups are excluded. The export:

  • Automatically fits all visible frames with a consistent margin.
  • Includes visible connection strings between frames.
  • Includes any active group border highlights.
  • Hides editing chrome (toolbar buttons, resize handles, pin icons).
  • Uses the dark #1a1d24 void background.

The PNG is rasterized using html2canvas and downloaded with the same filename stem as the project JSON.

Import

Click Import in the top bar and pick a .json file previously exported from Loom. Importing replaces the current board entirely. The app:

  • Switches to View Mode automatically after import.
  • Restores group hidden and border states from the file.
  • Resets the undo/redo history and records the imported state as the starting point.
  • Runs Zoom to Fit to bring all visible frames into view.
⚠️

Importing overwrites your current board and cannot be undone via Ctrl+Z. Export your current project first if you want to keep it.
